# AI招商智能体 MVP DEMO Guide

## 最高依据

本次 MVP 以根目录文件 `AI招商智能体 MVP计划.pdf` 为最高优先级依据。

如果现有代码、`original V0`、历史 PRD、README、原型页面或任何实现方案与该文件冲突，均以该文件为准。

## 当前核心理解

本项目目标不是做一个招商信息平台，也不是做一个提示词版 GPT。

目标是在 48 小时冲刺内完成一个可演示、可体验、可验证价值的 AI Native 招商智能体 MVP，让领导和客户在 5 分钟内感受到：

> AI 正在帮助我完成招商工作。

用户应感觉自己正在与一位资深招商顾问协同工作，而不是在操作一个后台系统或查询资料库。

## 产品定位

产品定位为：

> AI Native 招商工作台（AI Workspace）

明确不是：

- CRM
- BI 看板
- 企业管理系统
- 政策门户
- 信息平台
- 数据后台
- 知识库

## 本次 MVP 只聚焦四个能力

1. 企业分析
   - 输出：《目标企业招商研判报告》
   - 必须包含：企业画像、主营业务、产业链位置、招商匹配度、风险分析、招商建议、行动建议、数据来源

2. 产业分析
   - 输出：《产业链招商机会分析报告》
   - 必须包含：产业链结构、龙头企业、关键环节、本地产业现状、招商机会点、招商优先级、风险提示、数据来源

3. 政策分析
   - 输出：《政策匹配分析报告》
   - 必须包含：政策摘要、适用对象、支持内容、申报条件、招商价值、推荐产业、推荐企业、数据来源

4. 招商材料生成
   - 支持生成：招商邀请函、企业拜访提纲、电话沟通话术、微信跟进话术、项目汇报材料、项目推进计划

## 真实能力与模拟数据边界

允许使用：

- 本地企业库
- 本地政策库
- 本地产业库
- 本地招商案例库
- 本地知识库

禁止使用：

- Mock API
- 前端硬编码结果
- 写死分析报告
- 写死企业结论
- 写死政策结论
- 写死产业分析

核心原则：

```text
用户输入
↓
前端调用真实后端
↓
后端读取本地数据库 / 本地知识库
↓
后端调用真实大模型
↓
大模型生成分析结果
↓
前端展示真实结果
```

如果能力不足，必须明确标记“当前能力暂不支持”，不能伪造结果。

## 后端冻结原则

当前后端进入冻结模式。

禁止：

- 重构 LangGraph
- 重构 RAG
- 新增复杂 Agent
- 修改数据库结构
- 修改核心业务流程
- 重写后端架构

仅允许：

- Bug 修复
- API 联调
- 错误处理
- 超时处理
- 来源引用
- 稳定性优化

任何后端修改必须说明：

- 修改原因
- 风险评估
- 影响文件
- 回滚方案
- 验证方法

## original V0 使用规则

`original V0` 是当前已开发项目的参考来源。

允许：

- 只读查看
- 复用思路
- 复制必要代码到 `MVP DEMO`
- 调用其中可运行能力

禁止：

- 直接修改 `original V0`
- 删除 `original V0`
- 重构 `original V0`
- 移动 `original V0` 目录结构

## 开发顺序

严格按以下顺序推进：

1. 能力盘点
2. 产品方案设计
3. 用户体验设计
4. 页面与交互设计
5. 前后端对接方案
6. MVP 开发
7. Demo 优化

未完成前置阶段，不进入后续开发。

## 输出要求

结果不能是 GPT 式长文本或聊天记录式输出。

最终体验应接近：

- 咨询报告
- 招商研判报告
- 决策建议书
- 行动计划书

必须体现：

> AI 完成工作，而不是用户查资料。
