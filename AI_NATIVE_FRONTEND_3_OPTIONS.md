# AI Native 前端设计三方案

## 当前理解

用户希望前端摆脱信息系统、看板、CRM、知识库形态。

核心要求：

- 有直接自然语言交互
- 不让用户在散落信息中自己找
- AI 主动找到关键信息
- 结果要有判断、有来源、有下一步行动
- 风格要高质量、有设计感、科技清新

## 市场调研提炼

本轮参考的是优秀产品的交互思想，而不是视觉照搬。

| 产品 | 可借鉴点 | 转译到招商智能体 |
|---|---|---|
| Linear | 低噪声、命令优先、上下文动作、专业效率感 | 不做复杂菜单，当前答案上直接触发下一步动作 |
| Perplexity | 答案优先、来源跟随答案、自然语言追问 | 先给招商判断，再展示来源和可追问入口 |
| Manus | 目标输入、Agent 分解任务、持续执行 | 用户给招商目标，AI 拆解成检索、研判、材料生成 |
| Raycast | 快捷命令、动作搜索、少界面高效率 | Cmd-K 式上下文动作层，快速生成材料和切换视角 |

# 方案 A：招商答案引擎

## 一句话

> 问一句，直接得到招商判断、关键依据、来源和下一步材料。

## 产品感觉

像 Perplexity 的答案引擎，但不是搜索问答，而是招商研判。

用户打开后看到的不是导航，不是数据，不是看板，而是一个高质感自然语言输入：

```text
未来芯片科技值得重点招商吗？
如果值得，帮我生成下一步沟通材料。
```

系统直接输出：

- 是否值得推进
- 为什么值得推进
- 哪些数据支持这个判断
- 风险是什么
- 下一步该做什么
- 可生成什么材料

## 页面结构

| 区域 | 内容 |
|---|---|
| 主输入 | 自然语言招商问题 |
| 答案区 | 一句话招商判断 |
| 理由区 | 关键依据，不超过 3-4 条 |
| 来源区 | 企业库、CRM、政策库、产业图谱、知识库 citation |
| 行动区 | 生成拜访材料、微信话术、汇报材料 |

## 交互重点

- 答案先于信息
- 来源贴着判断
- 用户可以自然语言继续追问
- 下一步动作跟着答案出现

## 优势

- 最不像信息系统
- 最适合 48 小时 MVP
- 用户 5 分钟内能理解产品价值
- 复用当前企业分析、政策、CRM、材料生成能力最直接

## 风险

- 对回答质量和结构要求高
- 必须防止变成普通聊天框
- 需要严格控制结果格式

## 适合场景

领导演示、客户初次体验、MVP 首屏。

# 方案 B：Agent 任务航线

## 一句话

> 用户给一个招商目标，AI 展示它如何一步步完成任务，并最终交付报告和材料。

## 产品感觉

像 Manus 的 Agent 执行线程。

用户不是看数据，而是看 AI 的执行路径：

```text
理解目标
↓
检索事实
↓
合成判断
↓
生成交付
```

## 页面结构

| 区域 | 内容 |
|---|---|
| 任务目标 | 用户输入的招商目标 |
| 执行航线 | AI 分解出的步骤 |
| 状态反馈 | 哪些步骤已完成、哪些正在执行 |
| 交付物 | 报告、拜访提纲、微信话术 |
| 人工确认 | 哪些信息需要人工复核 |

## 交互重点

- AI 不是回答问题，而是在完成任务
- 用户看到可追踪的执行过程
- 最终交付物自然出现

## 优势

- Agent 感强
- 容易打动领导
- 能体现后端真实链路

## 风险

- 比方案 A 稍复杂
- 若执行步骤过多，可能又变成流程看板
- 需要把过程控制得很克制

## 适合场景

Demo 演示中段，用于证明 AI 不是静态问答，而是在完成招商工作。

# 方案 C：清流命令工作台

## 一句话

> 所有功能隐藏在上下文命令里，用户在当前答案上直接继续操作。

## 产品感觉

像 Linear + Raycast 的专业效率工具。

用户看到的是一份清爽的 AI 研判结果。  
当需要下一步动作时，通过命令层触发：

```text
生成拜访提纲
转成领导汇报材料
只看政策匹配
列出待确认风险
生成微信跟进话术
```

## 页面结构

| 区域 | 内容 |
|---|---|
| 当前答案 | AI 招商研判 |
| 命令层 | 上下文动作搜索 |
| 动作结果 | 材料、汇报、追问、复核清单 |
| 极简历史 | 最近完成的任务 |

## 交互重点

- 不靠菜单导航
- 不靠看板信息密度
- 当前上下文决定可用动作
- 招商经理可以高频使用

## 优势

- 专业感最强
- 适合长期产品化
- 操作效率高

## 风险

- 首次演示不如方案 A 直观
- 需要用户理解命令式操作
- 对细节交互要求高

## 适合场景

招商经理日常使用、二期增强、高频工作流。

# 推荐选择

## MVP 首选

> 方案 A：招商答案引擎。

原因：

- 最符合用户反馈：不看板、不散落、不让用户找信息
- 最能体现自然语言交互
- 最容易在 48 小时内落地
- 最适合领导 5 分钟理解

## 最佳融合

建议最终产品采用：

```text
方案 A 作为主体验
方案 B 作为 AI 执行过程动画
方案 C 作为隐藏高级操作层
```

即：

```text
自然语言提问
↓
AI 任务航线短暂展示执行过程
↓
招商答案引擎输出判断和来源
↓
命令层继续生成材料 / 汇报 / 风险清单
```

## 风格建议

统一采用科技清新风格：

- 大面积浅色留白
- 薄荷绿 / 青蓝 / 石墨黑作为主色
- 半透明玻璃质感
- 少量柔和光晕
- 高级但不厚重
- 不使用传统后台侧边栏
- 不使用 KPI 看板首页
- 不使用大面积深色科技大屏
