# V4 古典简约 AI Native 前端方案

## 当前理解

用户认可上一版在 UI 质感上已有明显进步，也认可“A+B+C 融合”的方向，但指出：

- 配色过于随意，不够正式
- 需要更古典、简约、可信
- 第一版 `02` 详页有可取之处
- V2 `02` 过程页有可取之处
- 需要从实用性、功能性、展示性继续加强
- 必须能在 5 分钟内让领导觉得这是值得花钱继续做的 AI Native 产品

因此 V4 不再追求“科技清新”的轻快感，而是升级为：

> 古典简约的 AI Native 招商研判工作台。

关键词：

- 正式
- 克制
- 高级
- 可信
- 咨询报告感
- AI 原生工作流
- 领导演示价值

## 2026-06-09 设计调研后的修正

用户反馈 V4 仍然偏厚重，科技感不足，需要更 AI Native。调研后决定保留古典简约底色，但增加“轻科技 AI 层”：

- 首页增加细线网格、AI 状态标签和轻量上下文信号
- 过程页增加 Agent 正在使用的数据源胶囊
- 研判页增加答案内联来源，而不是只放右侧证据栏
- 材料包改称 Artifact 材料包，强调可继续生成、改写、导出
- AI 执行中的当前步骤增加轻微脉冲状态，避免传统系统感

新的视觉目标：

> 古典简约底色 + 轻科技 AI 状态层 + 答案优先交互。

## 2026-06-09 AI Native 海外案例再调研后的 V4.1 修正

本轮参考国外 AI Native 产品的共性模式，而不是照搬视觉外观：

- Manus / GitHub Copilot Cloud Agent：AI 不只是回答，而是能拆任务、建计划、调用工具、产生可审查的工作痕迹。
- Claude Artifacts / v0 / Replit Agent：高价值输出应该进入独立 Artifact 区域，可继续编辑、迭代、导出，而不是停留在聊天气泡。
- ChatGPT Projects：AI 需要明确上下文空间，能围绕项目资料、指令和历史持续工作。
- Linear：交互应以命令和行动为中心，减少用户在多个信息区块之间寻找入口。

因此 V4.1 在本地原型中新增：

- 首页增加 `Agent loop`：Plan / Retrieve / Reason / Artifact，让第一屏直接体现代理式工作流。
- 首页增加 `AI 已理解` 与 `预计输出`，让自然语言输入不再像普通搜索框，而像任务委托入口。
- 执行页增加 `agent-runline`：显示 plan → tool calls → grounded reasoning → artifact 的运行态。
- 视觉上加入轻量扫描线、运行状态、技术青色细线，但保持正式、克制、可用于领导演示。

新的判断标准：

> 用户看到的第一感受应该是“我把招商任务交给一个会做事的 AI 代理”，而不是“我进入了一个招商信息系统”。

## 设计基调

### 风格

古典简约，不做“科技大屏”，不做“后台看板”，不做“花哨 AI 玻璃风”。

整体应像：

- 高端咨询报告
- 政务级智能工作台
- 研究院式决策桌面
- 轻量 AI Agent 产品

### 配色

建议采用稳定、正式、耐看的低饱和体系：

| 角色 | 颜色 | 用途 |
|---|---|---|
| 象牙纸白 | `#F7F1E6` | 页面底色 |
| 宣纸白 | `#FCFAF5` | 内容面板 |
| 墨色 | `#1B1915` | 主文字、主按钮 |
| 青玉绿 | `#2E6655` | AI 状态、可信来源、成功状态 |
| 铜金 | `#8B5E34` | 强调、章节标识、权威感 |
| 石青灰 | `#354E5C` | 专业说明、辅助信息 |
| 细线米灰 | `#D9CCB8` | 分割线、边框 |

### 视觉规则

- 少用大面积彩色
- 少用渐变
- 少用发光
- 少用卡片堆叠
- 多用留白、细线、纸感、章节感
- 把 AI 的“过程”做得可信，而不是炫
- 把结果做成“可交付件”，而不是聊天记录

# V4 推荐产品结构

## 总体体验

```text
自然语言立项
↓
AI 工作痕迹
↓
招商研判详页
↓
行动材料包
↓
领导演示价值页
```

这是一条完整的 5 分钟演示路径。

它不是普通页面集合，而是一套 Demo 剧本：

1. 领导看到自然语言入口，理解“不是后台”
2. 领导看到 AI 正在检索和分析，理解“不是假结果”
3. 领导看到正式研判报告，理解“能用于招商决策”
4. 领导看到材料包生成，理解“能减少实际工作量”
5. 领导看到价值总结，理解“值得继续投入”

# 页面 01：自然语言招商研判桌

## 目标

让用户第一眼理解：

> 我不用找功能，我只要说招商任务。

## 页面内容

主区域只保留：

- 产品名称
- 一句话定位
- 大号自然语言任务输入框
- 少量任务示例
- AI 可调用的数据来源说明

## 示例输入

```text
分析未来芯片科技是否值得重点招商；
如果值得，生成今天下午拜访用的提纲和微信跟进话术。
```

## 设计重点

- 不放左侧菜单
- 不放 KPI
- 不放企业表格
- 不放政策列表
- 不放“功能卡片”

首页应像一张安静的“招商研判桌”，而不是系统首页。

## 实用性加强

输入框下方只放 5 个高频任务：

- 研判一家企业
- 找本周优先企业
- 匹配政策抓手
- 生成招商材料
- 转领导汇报

这些不是入口卡片，而是可插入输入框的任务模板。

# 页面 02：AI 工作痕迹

## 继承点

保留 V2 `02` 过程页的优点：

- 用户能看到 AI 正在做事
- 过程不是普通 loading
- 展示真实检索与分析链路

## 目标

让领导相信：

> 这个结果不是前端写死的，也不是模型随口编的。

## 页面内容

以单一任务线程展示：

```text
识别任务
命中企业
读取上下文
形成判断
生成交付
```

每一步展示：

- 当前状态
- 使用的数据源
- 是否完成
- 是否需要人工确认

## 示例

| 步骤 | 显示内容 |
---|---|
| 识别任务 | 企业招商研判 + 沟通材料生成 |
| 命中企业 | 企业库命中：未来芯片科技有限公司 |
| 读取上下文 | CRM、风险评分、政策库、产业图谱、知识库 |
| 形成判断 | 补链价值高，风险可控，建议重点推进 |
| 生成交付 | 研判报告、拜访提纲、微信话术 |

## 设计重点

古典简约版本中，过程页不要做成炫酷流程图。  
应像“审计轨迹”或“研究过程记录”。

关键词：

- 细线
- 时间轴
- 状态章
- 数据来源
- 可验证

# 页面 03：招商研判详页

## 继承点

保留第一版 `02 Living Due-Diligence Brief` 的优点：

- 报告感强
- 结构正式
- 结论和来源并列
- 领导容易信任

## 目标

让结果不像聊天记录，而像可以给领导看的招商研判报告。

## 页面结构

```text
左侧：报告章节
中间：核心判断与正文
右侧：证据来源
底部：下一步行动
```

## 报告章节

- 一、核心判断
- 二、企业画像
- 三、产业链位置
- 四、政策匹配
- 五、风险与待确认
- 六、行动建议

## 核心展示

```text
核心判断：
值得重点推进

未来芯片科技可作为本周重点招商对象。
其 AI 推理芯片方向能补强园区半导体与人工智能交叉链条，
且可匹配高企免租、AI 产业扶持与算力资源政策。
```

## 关键指标

注意：指标不是 KPI 看板，而是报告里的研判标签。

- 匹配度：86/100
- 风险等级：低风险
- 推荐动作：安排拜访

## 证据来源

右侧来源不做成数据列表，而做成“证据栏”：

- 企业库：融资、规模、专利、行业
- CRM：历史沟通与关注点
- 政策库：高企免租 / AI 扶持
- 产业图谱：补链缺口与机会
- 知识库：入驻材料与案例

# 页面 04：材料包与上下文行动

## 目标

让领导看到产品不只是分析，还能把分析变成实际招商动作。

## 页面内容

基于当前研判，系统自动建议：

- 生成 20 分钟拜访提纲
- 生成微信跟进话术
- 转成领导汇报材料
- 生成风险复核清单
- 生成政策匹配摘要

## 命令层

借鉴 Linear / Raycast 的命令思想，但语义必须招商化。

命令不是：

```text
打开页面
搜索数据
筛选表格
```

而是：

```text
把这份研判转成领导汇报
只看政策匹配依据
列出还需要人工确认的问题
生成给企业负责人的微信话术
```

## 实用性加强

每个材料都应满足“可直接拿去用”：

- 标题明确
- 对象明确
- 语气明确
- 可复制 / 可导出
- 保留来源与依据

# 页面 05：领导演示价值页

## 目标

5 分钟 Demo 的最后一屏，让领导理解：

> 这个产品值得继续投入。

## 页面内容

不是功能列表，而是价值总结：

```text
以前招商经理需要：
查企业、翻政策、问产业链、写材料。

现在只要一句自然语言：
AI 在 2-3 分钟内给出可追溯研判和可直接使用的招商材料。
```

## 三个价值点

| 价值 | 说明 |
|---|---|
| 实用性 | 企业研判、政策匹配、产业机会、材料生成形成闭环 |
| 功能性 | 自然语言入口、真实后端链路、来源引用、上下文命令层 |
| 展示性 | 5 分钟可演示：输入任务、AI执行、输出报告、生成材料 |

## 领导能感受到什么

- 这不是资料系统
- 这不是 GPT 壳
- 这是能改变招商工作方式的 AI 工具
- 现有后端能力已经能支撑 MVP
- 下一阶段值得继续投入

# 5 分钟演示脚本

## 第 0-1 分钟：自然语言发起任务

打开产品，不介绍菜单，直接输入：

```text
分析未来芯片科技是否值得重点招商；
如果值得，生成今天下午拜访用的提纲和微信跟进话术。
```

讲解点：

> 过去要查多个系统，现在只需要说招商任务。

## 第 1-2 分钟：展示 AI 工作痕迹

展示 AI 正在：

- 命中企业
- 读取 CRM
- 查询风险
- 匹配政策
- 分析产业链

讲解点：

> 这不是前端写死结果，后面连接的是本地企业库、政策库、CRM、产业图谱和大模型。

## 第 2-3 分钟：展示招商研判详页

展示：

- 值不值得推进
- 为什么值得推进
- 来源在哪里
- 风险是什么
- 下一步建议是什么

讲解点：

> 输出不是聊天记录，而是一份招商研判报告。

## 第 3-4 分钟：生成材料包

继续生成：

- 拜访提纲
- 微信跟进话术
- 领导汇报摘要

讲解点：

> AI 不只是分析，还能把分析变成招商动作。

## 第 4-5 分钟：总结价值

展示价值页。

讲解点：

> 这个 MVP 已经能验证：AI 可以进入招商实际工作流，而不是停留在知识问答。
