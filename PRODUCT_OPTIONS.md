# 三套 AI Native 产品方案

## 当前理解

MVP 不是把 `original V0` 的 CLI Agent 套一个普通后台页面。

这次 Demo 必须让用户在第一眼感受到：

> AI 正在替招商人员完成研判、判断、建议和材料生成。

因此产品方案应围绕“任务完成”设计，而不是围绕“数据浏览”设计。

## 能力盘点结论引用

根据 `CAPABILITY_AUDIT.md`，当前最适合优先包装的真实能力是：

| 能力 | 成熟度 | 适合 Demo 程度 |
|---|---|---|
| 企业分析 | 基本可用 | 最高 |
| 招商材料生成 | 基本可用 | 最高 |
| 产业分析 | 部分可用 | 高 |
| 政策分析 | 部分可用 | 中高 |
| RAG / MCP | 代码存在，需环境验证 | 中 |

因此三套方案都应优先围绕“企业分析 + 行动材料生成”构建第一条 Demo 主链路。

# 方案 A：AI 招商任务驾驶舱

## 设计理念

用户不是进入一个系统，而是进入一个“招商任务执行台”。

首页只有一个核心动作：

> 输入招商任务，AI 自动完成研判。

系统用过程化界面展示 AI 正在做什么：

```text
识别任务
↓
检索企业
↓
读取 CRM
↓
分析风险
↓
匹配产业链
↓
匹配政策与资源
↓
生成招商研判报告
↓
生成下一步行动材料
```

## 用户旅程

1. 用户打开首页。
2. 输入：“分析未来芯片科技是否适合招商，并生成拜访提纲。”
3. AI 展示任务执行进度。
4. 系统左侧显示数据来源和工具调用过程。
5. 中间输出《目标企业招商研判报告》。
6. 右侧输出“下一步行动建议”和“一键生成材料”。
7. 用户点击生成拜访提纲、微信跟进话术、招商邀请函。

## 页面结构

| 区域 | 内容 |
|---|---|
| 顶部 | AI 招商任务输入框 |
| 左侧 | 执行步骤、工具调用、数据来源 |
| 中央 | 结构化招商研判报告 |
| 右侧 | 行动建议、材料生成、风险提示 |
| 底部 | 生成记录、可导出材料 |

## 核心交互

- 任务输入框不是普通搜索框，而是“招商任务指令台”。
- AI 执行过程分阶段展示，不只显示 loading。
- 每一段报告都能看到来源：企业库、CRM、政策库、产业图谱、知识库。
- 研判报告完成后，自动推荐下一步材料。
- 材料生成与研判结果联动，而不是单独写提示词。

## 优势

- 最符合“AI 正在帮助我完成招商工作”。
- 最容易复用现有企业分析和话术生成能力。
- Demo 节奏强，5 分钟内能看懂价值。
- 技术实现相对稳，不需要复杂多页面。

## 风险

- 对后端接口包装有要求。
- 如果 LLM 响应慢，需要做好过程态和超时提示。
- 报告结构必须强约束，否则容易变回聊天文本。

## 为什么适合招商场景

招商工作本质上是任务驱动：

- 判断企业值不值得跟
- 为什么值得跟
- 风险是什么
- 怎么推进
- 给谁发什么材料

方案 A 直接围绕这些工作结果组织体验。

# 方案 B：招商研判报告工厂

## 设计理念

用户进入系统后看到的不是聊天框，而是一个“报告生成工作流”。

用户选择报告类型：

- 目标企业招商研判报告
- 产业链招商机会分析报告
- 政策匹配分析报告
- 招商材料包

AI 根据输入对象生成咨询式报告。

## 用户旅程

1. 用户选择报告类型。
2. 输入企业、产业或政策关键词。
3. AI 自动检索数据。
4. 系统生成结构化报告。
5. 用户在报告侧栏中选择生成配套材料。
6. 最终导出报告或材料包。

## 页面结构

| 区域 | 内容 |
|---|---|
| 首页 | 四个报告入口 |
| 报告配置区 | 企业/产业/政策输入、报告深度、材料类型 |
| 生成过程区 | 数据检索、分析、生成进度 |
| 报告阅读区 | 类咨询报告结构 |
| 侧栏 | 数据来源、风险、下一步动作 |

## 核心交互

- 用户以“我要生成什么报告”为入口。
- 报告章节固定，避免 GPT 式长文本。
- 每个章节可以展开来源。
- 报告完成后生成材料包。

## 优势

- 和 MVP 计划要求的四类输出高度对齐。
- 结果非常正式，适合领导和客户看。
- 容易形成可交付感。

## 风险

- 体验可能偏“报告系统”，AI Native 感略弱。
- 如果首页做成四个入口卡片，容易像信息平台。
- 对四类报告能力要求更均衡，48 小时压力更大。

## 为什么适合招商场景

招商部门经常需要：

- 企业研判
- 产业分析
- 政策解释
- 汇报材料

方案 B 能把 AI 价值包装成“减少写报告和做材料的时间”。

# 方案 C：AI 招商作战室

## 设计理念

把招商工作设计成一个作战室：

- 目标企业池
- 产业链缺口
- 政策武器库
- 跟进动作
- AI 顾问建议

用户看到的是招商全局局势，AI 在旁边持续给出判断和行动建议。

## 用户旅程

1. 用户进入作战室。
2. 左侧看到重点企业线索。
3. 中间看到产业链地图和缺口。
4. 右侧 AI 顾问提示“今天最值得推进的 3 家企业”。
5. 用户选择企业。
6. AI 生成企业研判、风险、政策匹配和下一步动作。
7. 用户把企业加入推进计划。

## 页面结构

| 区域 | 内容 |
|---|---|
| 左侧 | 重点企业线索、筛选条件 |
| 中央 | 产业链地图、招商机会点 |
| 右侧 | AI 顾问、风险提醒、行动建议 |
| 底部 | 推进计划、待生成材料 |

## 核心交互

- 点击产业链缺口，AI 推荐企业。
- 点击企业，AI 输出研判。
- 点击政策，AI 推荐适用企业。
- AI 自动形成“今日招商优先级”。

## 优势

- 视觉冲击力强。
- 很适合领导看“大局”和“机会”。
- 能展示产业链招商的高级感。

## 风险

- 48 小时内实现复杂度最高。
- 产业图谱、企业推荐、政策匹配都要稳定。
- 容易做成 BI 看板或数据驾驶舱，偏离 AI Native。

## 为什么适合招商场景

招商领导关心的不只是单个企业，而是：

- 产业链哪里缺
- 哪些企业最值得引
- 政策如何形成招商抓手
- 团队下一步该打哪里

方案 C 对领导视角最友好，但实现风险较高。

# 推荐方案

## 推荐选择：方案 A，AI 招商任务驾驶舱

理由：

| 维度 | 方案 A | 方案 B | 方案 C |
|---|---|---|---|
| 48 小时落地 | 高 | 中 | 低 |
| 打动领导 | 高 | 中高 | 高 |
| AI Native 感 | 最高 | 中 | 高 |
| 复用现有能力 | 最高 | 中高 | 中 |
| 技术风险 | 中 | 中 | 高 |
| 符合 MVP 文件 | 高 | 高 | 中高 |

方案 A 是最稳的 48 小时 MVP 方案。

它不会把产品做成 CRM、BI 或知识库，而是让用户清楚看到：

> 我给 AI 一个招商任务，AI 帮我完成研判、判断、建议和材料生成。

## 推荐 Demo 主链路

```text
用户输入：
分析“未来芯片科技有限公司”是否适合招商，并生成拜访提纲和微信跟进话术。

AI 执行：
1. 读取企业画像
2. 查询 CRM 跟进记录
3. 查询风险评分
4. 分析产业链位置
5. 匹配园区政策和资源
6. 形成招商研判
7. 输出行动建议
8. 生成招商材料

最终输出：
《目标企业招商研判报告》
《企业拜访提纲》
《微信跟进话术》
```

## 推荐首页第一屏

第一屏不做传统导航首页。

建议结构：

| 区域 | 内容 |
|---|---|
| 主标题 | AI 招商智能体 |
| 副标题 | 输入一个招商任务，AI 完成研判、建议与材料生成 |
| 主输入 | “分析某企业是否适合招商，并生成下一步材料” |
| 快捷任务 | 企业研判、产业机会、政策匹配、生成材料 |
| 任务进度 | 识别、检索、分析、研判、生成 |
| 结果预览 | 咨询报告式结构，而不是聊天气泡 |

## 推荐视觉方向

关键词：

- 专业
- 高级
- 可信
- 智能
- 决策感

不建议：

- 大面积营销 hero
- 卡片堆砌后台
- CRM 菜单
- 数据看板式 KPI 首页
- 聊天记录式主界面

建议：

- 克制的深浅对比
- 报告式排版
- 清晰的任务执行流
- 数据来源标签
- 行动建议侧栏
- 结果区域像“咨询顾问交付物”

## 下一步计划

如果确认采用方案 A，下一阶段进入：

1. 用户体验设计
2. 页面结构设计
3. 前后端对接方案
4. MVP 开发

如果不确认，我可以基于方案 B 或 C 继续调整。
